Diferencias entre desarrollador front-end, back-end y full-stack

Un desarrollador front-end es aquel que se especializa en los lengajes de programación web que se ejecutan en la parte del cliente o del navegador. Algunos de estos lenguajes son:

  • HTML
  • CSS
  • JavaScript

Un desarrollador back-end es aquel que se especializa en los lengajes de programación web que se ejecutan en la parte del servidor. Algunos de estos lenguajes son:

  • PHP
  • NodeJS
  • Java
  • Python

Un desarrollador full-stack es aquel que se especializa en los lengajes de programación que se ejecutan tanto en el front-end como en el back-end.

Frameworks de CSS

Un framework es un entorno de trabajo que facilita las labores de programación, y entre sus ventajas destacan:

  • Aceleración del proceso de programación
  • Favorecimiento del trabajo colaborativo
  • Reducción de errores

Framework Bootstrap

Bootstrap es un framework del lenguaje CSS que permite dar estilos a los distintos elementos de una página web (títulos, botones, menús...). Actualmente se encuentra en su versión 5.1.1. Se puede descargar desde su página web o se puede enlazar directamente desde la cabecera (head) de nuestro sitio web.

Alternativas a Bootstrap

Actualmente, los principales frameworks CSS alternativos a Bootstrap son:

  • Tailwind
  • PaperCSS
  • Materialize CSS
  • Bulma

WCAG

Son las iniciales de Web Content Accessibility Guidelines (Directrices de Accesibilidad para el Contenido web). Son unas directrices recomendades (algunas de ellas de obligado cumplimiento en algunos países) que deben seguirse para hacer un sitio web accesible a personas con discapacidad. Estas directrices están marcadas o mantenidas, por decirlo de alguna forma, por el W3C Consortium

La importancia de validar en XHTML, CSS y WCAG 2.1 AAA se debe a que, al estar validadas, sabemos que nuestros sitios web son estrictos tanto a nivel de código como a nivel de accesibilidad. Pulsa aquí para ir al validador del W3C Consortium.

Servidores web y HTTP

Servidores web

Un servidor web es un software que permite a equipos remotos tales como pc's, smartphones, tablets, smart tv's... (equipos "cliente") conectarse al equipo "servidor" donde está instalado el software. Este equipo suele ser un ordenador, aunque otros dispositivos, como los mencionados anteriormente, pueden ser usados como servidor. Por tanto podemos entender el concepto de servidor de dos maneras:

  • Software específico que permite compartir información con equipos remotos
  • Equipo o dispositivo donde se encuentra instalado este software

HTTP

El protocolo HTTP(HyperText Transfer Protocol) es un protocolo que permite realizar peticiones de datos y recursos (normalmente una página web HTML con sus imágenes, enlaces, hojas de estilo CSS, scripts...) a un servidor.
Está basado en el modelo, estructura o arquitectura cliente-servidor (el cliente solicita la información al servidor, el servidor recibe la petición, la envía o sirve al cliente y el cliente la interpreta y la muestra al usuario que realiza la petición).
Este protocolo es la base de cualquier intercambio de datos en la red.

Métodos GET Y POST

Existen dos métodos que nos permiten intercambiar información entre cliente y servidor.

Método GET

Básicamente, este método expone en la URL la información validada en un formulario, y el sitio web que sirve esa información se comportará de una forma u otra en función de los valores enviados en el formulario y mostrados en la URL. Una caracterísitca importante de este método es que no es necesario reenviar el formulario para obtener los datos solicitados al servidor. Es útil en casos en los que NO enviamos datos sensibles (identidad, vivienda, datos bancarios...) al servidor, como la paginación de registros.

Un claro ejemplo de este método es esta página web de un curso de programación PHP. Si en la URL que nos aparece(https://www.tutorialesprogramacionya.com/phpya/index.php?inicio=0) cambiamos el valor de inicio a 10 (https://www.tutorialesprogramacionya.com/phpya/index.php?inicio=10), nos llevará hasta aquí, siendo el mismo sitio web pero mostrándonos otro contenido diferente.

Otro ejemplo es este generador de tablas de multiplicar

Método POST

Este método se suele utilizar para enviar al servidor datos sensibles (identidad, vivienda, datos bancarios...) y en ningún momento son visibles para el usuario. Al contrario que en el método GET, estos datos no aparecen en la URL. Para volver a validar datos en el servidor, en este caso, sí que sería necesario volver a enviar el formulario. Cualquier formulario de registro sería un claro ejemplo de uso de este método. Por ejemplo, el formulario de registro de la red social Twitter

Apache

Apache es un software de servidor HTTP, desarrollado y mantenido por la Apache Software Foundation

Es considerado el servidor nº1 en Internet. Es multiplataforma (Windows, OSX y Linux) y cuenta además con algunos módulos adicionales, como el módulo PHP, que es capaz de interpretar el lenguaje PHP, que es el que estamos estudiando en esta asignatura.

XAMPP

XAMPP es una pila (stack) o conjunto de programas compuesto por:

  • Apache (servidor web)
  • MySQL o MariaDB (gestor de bases de datos relacionales)
  • PHP (Lenguaje de programación de guiones)
  • Pearl (Lenguaje de programación, actualmente en desuso)

Todos estos elementos pueden instalarse por separado según las necesidades de la aplicacion que vayamos a alojar (no necesitaríamos PHP, por ejemplo, ni base de datos, para una página web de contenido estático). La pila XAMPP cuenta con un instalador que instala todas las soluciones anteriormente indicadas y cuenta con un panel de administración para gestionar los diferentes servicios instalados con esta pila.
